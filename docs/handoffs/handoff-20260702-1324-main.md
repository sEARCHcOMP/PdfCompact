## Gotchas / 注意点

- **SDK は静的 script タグにしない**。タブ初回クリックで `createElement('script')` 注入（firebase-app-compat → firebase-database-compat → qrcode-generator の直列）。静的にすると全ユーザーの起動が遅くなり、オフライン初回表示も壊れる
- `src/app/*.js` に **`</script>` の文字列を書くとビルドが落ちる**（build.js のガード）。動的注入なら自然に回避
- **ワーキングコピーは CRLF**。エディタ外のツールでファイルを作った/編集したら PowerShell の `(Get-Content $f) | Set-Content $f` で正規化してからコミット
- 転送プロトコル: RTDB `rooms/{roomId}/transfers/{fileId}` に `{name, safeName, size, mime, key, from: 'phone'|'pc', thumb?}`。ファイル実体は R2 `transfers/{roomId}/{fileId}/{name}`、Worker の `/download?key=` `/delete?key=` で取得/掃除。**受信後は必ず R2 + RTDB 両方を掃除**（60-hikari.js の `cleanupRemote`）
- スマホ在席は `rooms/{roomId}/phonePresence` を watch（接続で波紋停止 + ステータス切替）
- 部屋はブラウザタブを閉じると `onDisconnect().remove()` で自動消滅
- ブラウザ版は「受信専用」。PC→スマホ送信は Electron 版（pc-toolkit）のみ
- pc-toolkit の PDFC webview は pages.dev を直読みしてるので、**このリポに push するだけで Electron アプリ側にも反映**される（アプリ内ではタブバー非表示のため、スマホ転送タブはアプリからは見えない=干渉なし）
