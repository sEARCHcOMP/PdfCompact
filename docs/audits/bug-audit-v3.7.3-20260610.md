## 🔴 致命 (2件)

### C1. [PDF編集] 単一PDFのページ並び替えが必ず失敗する
- L11737 `outDoc.movePage(outDoc.getPage(curPos), i);` — **pdf-lib 1.17.1 に movePage は存在しない** (CDN実物をgrep、定義0件。L11723のコメント「pdf-lib provides this」は誤り)。
- 発動条件: ファイル1つだけ投入 (戦略A) + 順序変更あり。`outDoc.movePage is not a function` の TypeError → 赤字エラーで出力ゼロ。並び替え後の「選択を取り出し」も同様。
- 複数ファイル投入時は copyPages 経路 (戦略B) で動くため、テストをすり抜けたと推定。
- 修正方針: 戦略Aでも順序変更がある場合は copyPages 方式に落とす、が最小。

### C2. [画像配置] 回転ページ (/Rotate 90/180/270) で配置画像の位置・向きが出力で狂う
- L16580-16583 の TODO コメントが自白: プレビューは pdf.js viewport (回転後) 座標、出力は page.getSize() (回転前) 座標で無補正。
- スマホスキャナ・複合機の横原稿スキャンPDFで、プレビュー完璧 → 出力でハンコが90度回って別の位置。警告なし。
- 同バンドル内の PDF編集 (L11716) と透かし (L8514) は getRotation 補正済み。imgPlace だけ未対応。

## 🟠 高 (11件)

### H1. [軽量化] OCRオン時、出力ページ寸法が約2倍に膨張
- L9745-9762: canvas→JPEG (DPI情報なし) を Tesseract に渡し、返却PDFページを無修正コピー。Tesseract は DPI 不明画像を 70dpi 扱い → 寸法 = px × 72/70。
- **Node + tesseract.js@5.1.0 で実測検証済み**: 1240×1754px 入力 → MediaBox 1275.43×1804.11pt (A4 が A2 超に)。
- OCRオフの写真モード (L9683-9686) は原寸復元しており、OCRオンだけ欠落。

### H2. [軽量化] 書類モード: グレースケール/CMYK JPEG の再圧縮で色化け (confidence: medium)
- L9599-9620: canvas 再エンコードJPEGは必ずRGB(3成分)なのに ColorSpace は元辞書をコピー → 仕様違反ストリーム。
- /DeviceGray (複合機スキャン挿入頁) や /DeviceCMYK で、pdf.js 系ビューア (Firefox内蔵等) で縞ノイズ・色化け。圧縮した本人の環境では正常に見えるのが厄介。

### H3. [画像→PDF] 用紙サイズ「画像合わせ」で横長写真が縦ページに化けて右1/4欠け+白帯
- L10419-10423: jsPDF コンストラクタ/addPage に orientation 未指定。jsPDF 2.5.1 実ソース確認: **幅>高さのページ指定はデフォルト 'p' で強制縦横スワップ**される。addImage は元寸で描くため右側欠落。
- A4/A3/B4/B5 プリセットは orientation 明示済みで無事。「画像合わせ」だけ壊れてる。

### H4. [画像→PDF] 画質「無圧縮」でスマホ縦撮り写真 (EXIF Orientation) が横倒し+歪み
- L10504-10506: 無圧縮だけ canvas を通さず生 dataURL を jsPDF 直渡し。jsPDF 2.5.1 に EXIF 処理0件 (実ソース確認)。さらに枠寸法は EXIF 補正後の naturalWidth/Height で計算するため、縦比率の枠に横倒しピクセル = 90度倒れ+アスペクト潰れの二重被害。
- 軽量/標準/高画質は canvas 経由で正常 → 「無圧縮だけ倒れる」不可解な差に見える。

### H5. [変換] 透過PNG/AVIF→PDF で背景が黒化け (FAQの約束と真逆)
- L10822-10827: 白埋めは JPG/BMP 出力時のみ。PDF出力は L11050-11051 で透明 canvas を toDataURL('image/jpeg') → 透過部が RGB(0,0,0)。TIFF入力経路は白埋め自体なし。
- FAQ L7892 は「JPG / PDF:透過は白背景に変換されます」と明言。印鑑・ロゴPNGのPDF化で背景真っ黒。

### H6. [変換] ZIP内の同名出力が無言上書き → 欠落するのに「✓ N枚変換」
- L11115 出力名=元名の拡張子差し替え、L11153-11154 重複チェックなしで JSZip.file() (同名は上書き)。スマホ2台の IMG_0001.JPG や IMG_0001.HEIC+IMG_0001.JPG ペアで後勝ち消失。完了表示は全数成功を主張。

### H7. [PDF編集] 読込失敗のエラーが即消去 → 欠落・部分読込のまま無言で結合
- L11378-11388: catch でエラー表示 → 直後の `setStatus('')` が無条件で消す。途中ページ失敗時は前半ページだけ残ってロールバックなし。契約書セットの1ファイル欠落が警告ゼロで完成扱い。

### H8. [PDF編集] 編集制限付き (オーナーパスワード) PDF が「成功」のまま壊れたPDFを出力
- L11646/L11689: `ignoreEncryption: true` で強行ロード。pdf-lib 1.17.1 に復号機能なし (実物確認) → 暗号化ストリームを再採番保存 = 白紙/文字化け/開けないPDF。プレビュー (pdf.js は透過復号できる) は正常に見えるため発見が遅れる。
- 同梱 PdfSanitize は L8598-8599 で真逆の安全策 (ignoreEncryption 付けない→原本返し) を明記。編集モードだけ無防備。

### H9. [画像配置] CropBox≠MediaBox の図面PDFで全配置が一定量ズレる
- 配置座標は pdf.js viewport (CropBox) 基準、出力は pdf-lib getSize() (MediaBox) + 原点(0,0)前提 (L16584-16591)。getCropBox/getMediaBox 呼び出し0件。CAD/プロッタ出力で頻出。

### H10. [共通基盤] 透かしON時、フォント取得失敗で透かし無しPDFが「✓完了」(無通知)
- L8398: 透かしフォントは実行時に jsdelivr から fetch。L8481-8483/L8488-8492: 失敗 catch → 無言 return (スキップ)。暗号化スキップには通知フラグ (L8625) があるのに透かしには無し。
- **オフライン/社内プロキシ遮断で「社外秘」が入らないまま配布** のリスク。設定画面に要ネット注記もなし。

### H11. [共通基盤] サニタイズ失敗時に未処理PDFが無通知で出る (フラグの配線ミス)
- L8614-8617: 加工/保存失敗 → 無言で原本 blob 返し (フラグもconsoleもなし)。
- L8603 の `_lastSkippedEncrypted` フラグを読むのは変換モードのみ (L11137) だが、変換は canvas 再構築なので暗号化出力は構造上発生しない = 死にチェック。実際に発生し得る軽量化の「元のまま」DL経路 (L9453-9457→L9332-9341) はフラグを見ない → メタ除去ON(既定)を信頼した利用者が作成者名入りPDFを配布。
